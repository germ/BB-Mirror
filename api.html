Art As A Service

Library & JSON API Reference.

Collage has the amazing ability to destroy any conception of a final work, the chaotic fragments re-arranging and ripping their own story into your canvas. There's a force inside these fragments that wants to be unlocked, the materials and I interrogate each other. Revealed thoughts from a opaque interface.

This is a pragmatic method to send me fragments, meant to be woven into running services or programs that occasionally run. My work is best executed when used as a logging service would with zero interaction. Over time, this establishes a context based off fragments of data. On triggers in your application works will simply arrive at the door when the code (Not you, or I) deem fit.

Looking to visualize the data from a stack of files? They've been sitting on disk for years, why not get them on the wall?

This project is a attempt at reconciling the divide between Artmaking process and Engineering sensibilities. f(x) -> [Object Art] if you will. Any input will suffice, f(ðŸ’©) will be evaluated!

Library

I've created a simplistic library for this API in Go, hosted on GitHub. The library allows uploading and managing sessions in a simple manner. Wrapping existing readers transparently or new instrumentation, adding BlackBox to your infrastructure only takes a few lines of set up!

Endpoint/Docs

Base URL: https://theblackbox.ca/api

Query Parameters

The Available parameters are

  • action: "create" or "finalize"
  • ID: Your unique ID used for authing requests, obtained below.
  • session: The works identifier
  • pay: The offered amount for the finished work

Response Object

  • Success: (Bool) describing if the request action was preformed sucessfully
  • Error: (String) contains an error (set only if Success is false)
  • ID: (String) Your user ID key used in request
  • Session: (String) The specfic work that the request refers to
  • Approved: (Bool) If this work has been reviewed and accepted/charged
  • Sessions: ([]String) List of all sessions for the account

Workflow

A typical work flow looks like this:

  • (Create Session ) A call to /api?action=create&ID=1234 (saving the Session var)
  • (Store dialogue ) POST requests to /api?session=7890&ID=abcd
  • (Payment Bid ) Call to /api?pay=12345&session=7890&ID=abcd
  • (Submit work req) A call to /api?action=finalize&session=7890&ID=abcd

/api?action=create (ID) Create returns a response object with a new Session
POST /api (ID, Ses) Stores the POSTed files to a given session
/api?action=finalize (ID, Ses, pay) Submit a session for approval


If you run into issues, compare against the Library implementation or simply contact me.
